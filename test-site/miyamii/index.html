★ CASHEW NETWORK PROTOCOL v0.1-alpha

Cashew Network is a custom peer-to-peer distributed network protocol and node implementation. The goal is a lightweight, extensible P2P infrastructure built from scratch without relying on existing frameworks like libp2p.

╔══════════════════════════════════╗ ║ CASHEW NETWORK TOPOLOGY ║ ╠══════════════════════════════════╣ ║ ║ ║ [Node A] ←──────→ [Node B] ║ ║ ↑ ↑ ║ ║ └──────→ [Node C] ─┘ ║ ║ ↑ ║ ║ [Node D] ... ║ ║ ║ ║ each node = peer + router ║ ╚══════════════════════════════════╝

// Architecture

The node binary is written in C++ and handles peer discovery, message routing, and protocol negotiation. A Perl demo server (server.pl) serves as a bootstrap/rendezvous point for new nodes joining the network.

// Why Cashew?

Because cashews are technically drupes, not nuts. Subverting expectations. Just like a high schooler building a custom network protocol. Also I just like the name. (ᵔᴥᵔ)

// Goals

  • Fully decentralized — no single point of failure
  • Low overhead, suitable for embedded / low-power nodes
  • Eventually: run on custom SeaBird hardware
★ SEABIRD ISA + CLIONE PROCESSOR research-grade

SeaBird is a custom hybrid CISC/RISC instruction set architecture (ISA). It powers a family of three custom processor designs named after sea creatures from YonKaGor's animations.

ProcessorTargetISANode
ClownfishPerformanceRISC-V130nm

ClioneThroughputSeaBirdWide superscalar

// Clione Architecture Highlights

[ CLIONE CHIP LAYOUT ] ┌─────────────────────────────────────┐ │ CLIONE DIE │ │ │ │ ┌──────────────┐ ┌─────────────┐ │ │ │ MAIN CORES │ │ TRANSLATION │ │ │ │ (SeaBird) │ │ CHIPLET │ │ │ │ │ │ x86/ARM/ │ │ │ │ Wide super- │ │ RISC-V │ │ │ │ scalar exec │ │ → SeaBird │ │ │ └──────┬───────┘ └──────┬──────┘ │ │ └────────┬─────────┘ │ │ ┌────────┴────────┐ │ │ │ CACHE-COHERENT │ │ │ │ INTERCONNECT │ │ │ │ (Infinity-style)│ │ │ └─────────────────┘ │ └─────────────────────────────────────┘

// Key Features

  • FET (Fast Execution Technology) — per-unit dynamic clock boosting, self-characterizes at boot time
  • Dynamic Binary Translation — run x86, ARM, and RISC-V code natively via translation chiplet
  • Cache-coherent chiplet interconnect — modeled on AMD Infinity Fabric
  • RISC-V native compat mode built into SeaBird encoding

// Toolchain

All verification done with Verilator + SystemVerilog (commercial EDA tools cost $$$ lol). Bootloader written in custom seabird_boot.asm — handles reset vector, GDT/IDT, paging, kernel handoff.

★ PELAGE :: Programmatic Environment for Lagomorphic and Anthropomorphic Garment Engineering intentionally over-engineered
